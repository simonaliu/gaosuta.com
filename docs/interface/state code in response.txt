0和负值代表通用值, 其定义在Java源代码中类org.varkrs.sociality.common.web.configurations.StateCodes介绍. 下面说明其具体含义:

0: 成功. 
-1: UNKNOW_ERROR. 
-2: 未登录. 
-3: 未授权. 即用户可能修改(更新, 删除等)一个不属于自己的资源(如日志, 图片等). 
-4: 资源不存在. 

在有些文档里看到state可能返回"其他"这个值, 一般来说, 只有指定的说明的返回值才会有意义, 如果你在调用的过程中出现了属于其他范围的值, 只会是下面两种情况: 
1. 后台代码里确实设置了这个值, 但文档没有说明(这种情况相当少见)
2. 出现了后台也不知道的异常情况, 属于bug. (这种情况更加少见)

关于未知错误的说明:
未知错误有两种含义, 
一是后台设置了state为-1, 这是后台表示前台需要的请求失败的一种状态码. 
二是后台设置了其他这个值, 这如上面的讨论, 后台设置了一个自己并没有意识到的值, 属于bug. 
一般来说, 未知错误在有些情况下并不是未知的. 其中更有可能属于数据库访问的"正常"的异常. 例如: 
1. 在进行SELECT操作的时候, 数据库返回null.  
2. 在进行UPDATE操作的时候, 数据库并没有被更新. 
因为后台的数据库接口实现是这样一个逻辑: 后台总是捕获所有的异常, 然后返回null或者false. 也就是说, 当进行SELECT操作的时候, 返回null会有两种解释: ①资源不存在; ②数据库抛出异常. 
所以当获取null值时, 前台不能主观臆测到底是资源不存在还是数据库抛出异常, 所以只好设置状态码为-1(即UNKNOW_ERROR). 这种情况也适合UPDATE操作的讨论.
